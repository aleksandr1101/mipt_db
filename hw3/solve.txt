Данные:
    файл: 20mb.json
    поля:
        name
        email
        address
        phone
        website

Создаем кластер:
    minikube start --cpus=4 --memory=4000 --nodes=4  --kubernetes-version v1.29.3
Получаем отказоустойчивый Redis кластер:
    kubectl get po -n redis                                            
Скопируем данные в локальный файл redis.conf из интернета или системного.
Поменяем в нем поле: timeout 10000
Применим новую конфигурацию к кластеру через ConfigMap:
    kubectl create configmap redis-config --from-file=redis.conf


Сохраняем данные в кластер в виде разных структур: 1.py
2_1 -- 2_4 измерение времени со string, hset, zset, list
